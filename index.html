4. Платежи и добровольные пожертвования

4.1. Природа платежей

Все платежи в пользу бота являются добровольными пожертвованиями (Donations) на развитие проекта. В благодарность за поддержку вы получаете виртуальную валюту, однако это не является коммерческой сделкой купли-продажи.

4.2. Обработка платежей через CryptoPay

  • Все платежи обрабатываются через стороннего платежного провайдера CryptoPay (CryptoBot API)
  • Бот интегрирован с CryptoPay API для создания платежных инвойсов и проверки статуса транзакций
  • Бот не хранит данные ваших банковских карт, кошельков или платежных реквизитов
  • При совершении платежа вы принимаете условия использования CryptoPay и их политику конфиденциальности
  • После успешного платежа виртуальная валюта (Звезды) зачисляется на ваш аккаунт автоматически и мгновенно
  • Каждая транзакция записывается в базу данных бота с уникальным идентификатором инвойса (Invoice ID)
  • Владелец бота не несет ответственности за технические сбои, задержки или ошибки платежной системы CryptoPay
  • В случае технических проблем с зачислением средств, обратитесь в поддержку с предоставлением Invoice ID

4.3. Политика возврата средств

ВНИМАНИЕ: Возврат средств (рефанд) не предусмотрен в связи со следующим:
  • Цифровая валюта начисляется мгновенно и автоматически после подтверждения платежа
  • Цифровой контент потребляется немедленно
  • Платежи являются добровольными пожертвованиями
  • В соответствии с законодательством о защите прав потребителей большинства юрисдикций, цифровой контент, предоставленный немедленно с согласия потребителя, не подлежит возврату

Исключения: Возврат может быть осуществлен только в следующих случаях технической ошибки:

  • Платеж был успешно проведен через CryptoPay, но Звезды не были зачислены на аккаунт
  • Произошло двойное списание средств за одну транзакцию
  • Технический сбой в API бота привел к незачислению валюты

Процедура обращения по техническим ошибкам:

  1. Обратитесь в службу поддержки через официальный сервер Discord или напрямую к владельцу бота
  2. Предоставьте доказательство платежа (скриншот подтверждения CryptoPay, Invoice ID, Transaction Hash)
  3. Укажите ваш Discord User ID и время совершения платежа
  4. Запросы обрабатываются в течение 3-7 рабочих дней

НЕ подлежат возврату:

  • Ситуации, когда пользователь передумал или пожалел о покупке
  • Потеря виртуальной валюты из-за собственных действий (передача другим пользователям, проигрыш в кейсах, продажа NFT)
  • Неудовлетворенность результатами открытия кейсов
  • Блокировка аккаунта за нарушение правил

4.4. Комиссии и сборы

Продажа NFT-подарков обратно в систему:

  • При продаже NFT-подарка через соответствующую функцию бота взимается системная комиссия в размере 15% от текущей рыночной стоимости предмета
  • Вы получаете 85% от стоимости NFT в виде виртуальной валюты (Звезды)

  • Проданный NFT удаляется из вашего инвентаря и возвращается в общий пул системы

Передача активов между пользователями (БЕЗ комиссии):

  • Минимальная сумма перевода Звезд может устанавливаться администрацией
  • Переводы между заблокированными аккаунтами запрещены

  • Размер комиссии на продажу NFT может быть изменен владельцем бота с уведомлением за 7 дней через официальные каналы

Для обеспечения функционирования бота мы собираем и храним следующую информацию:

  • Идентификатор пользователя Discord (User ID): Уникальный числовой идентификатор, присваиваемый Discord
  • Идентификаторы серверов (Guild ID): ID серверов Discord, где вы используете бота
  • Публичное имя пользователя (Username): Отображаемое имя для таблиц лидеров и уведомлений
  • Аватар пользователя (Avatar URL): Для визуального отображения в интерфейсе

2.2. Игровые данные

Бот хранит следующие игровые данные в SQLite базе данных:

Профиль пользователя (таблица users):

  • Баланс виртуальной валюты "Звезды": Текущее количество Звезд на вашем счете
  • Настройки уведомлений: Ваши предпочтения по получению уведомлений от бота
  • Язык интерфейса: Выбранный язык (русский, английский и другие поддерживаемые локали)

Инвентарь NFT (таблица inventory):

  • Список NFT-подарков: Все NFT-предметы, которыми вы владеете
  • Уникальные серийные номера: Каждый экземпляр NFT имеет уникальный серийный номер (serial_number)
  • ID экземпляра: Уникальный идентификатор каждого конкретного NFT в вашем инвентаре (instance_id)
  • Связь с профилем: Привязка NFT к вашему Discord User ID

Магазин и цены (таблица market):

  • Текущие цены всех NFT-подарков
  • Количество доступных экземпляров лимитированных подарков (stock)
  • Исходное максимальное количество лимитированных подарков (initial_stock)

История транзакций внутри бота:

  • Полная история всех операций с виртуальной валютой (получение, расходование, переводы другим пользователям)
  • Логи передачи Звезд между пользователями с указанием отправителя, получателя, суммы и времени
  • История покупок NFT-подарков в магазине
  • История продажи NFT обратно в систему
  • История передачи NFT другим пользователям

Кейсы (таблица cases):

  • Количество доступных бесплатных кейсов
  • Количество кейсов за Звезды
  • История открытия кейсов и полученных из них наград

Ежедневные награды (таблица cases):

  • Время последнего использования команды /cases
  • Отслеживание 24-часового кулдауна

Реферальная система (таблицы referral_codes и referral_history):

  • Ваша реферальная ссылка: Уникальный код приглашения, сгенерированный для вашего аккаунта
  • Список приглашенных пользователей: User ID всех пользователей, которые присоединились по вашей реферальной ссылке
  • История использования кодов: Информация о том, чей реферальный код вы использовали при регистрации
  • Дата создания кода: Timestamp создания вашей реферальной ссылки
  • Полученные реферальные бонусы: Записи о Звездах, полученных за приглашение новых пользователей

2.3. Платежные данные

ВАЖНО: Бот НЕ собирает и НЕ хранит:

  • Данные банковских карт
  • Пароли от криптокошельков
  • Seed-фразы или приватные ключи
  • Платежные реквизиты

Мы получаем от платежного провайдера CryptoPay и храним следующие данные:

База данных транзакций (таблица transactions):

  • Invoice ID (идентификатор инвойса): Уникальный идентификатор платежа, сгенерированный CryptoPay
  • User ID: Ваш Discord User ID для привязки платежа к аккаунту
  • Сумма пополнения (amount): Размер платежа в валюте CryptoPay
  • Статус платежа (status): Текущий статус транзакции (pending - ожидание, paid - оплачено, cancelled - отменено, expired - истекло)
  • Дата и время создания (created_at): Timestamp создания платежного инвойса

Индивидуальные логи платежей (директория payLogs/):

  • Для каждого пользователя создается отдельный лог-файл с именем {user_id}.log
  • Записи в логах содержат:
    • Временные метки всех событий платежа (московское время)
    • Тип события (создание инвойса, проверка статуса, успешное начисление, ошибки)
    • Детали транзакции (Invoice ID, сумма, количество начисленных Звезд)
    • Индикаторы успеха (✅ INFO) или ошибок (❌ ERROR)
    • Технические детали для диагностики проблем
  • Доступ к логам: Логи платежей доступны для просмотра администраторами (разработчик и служба поддержки) через команду /admin getlogs для решения технических проблем

Логи переводов Звезд (директория logs/):

  • Логи всех переводов виртуальной валюты между пользователями
  • Записываются: ID отправителя, ID получателя, сумма перевода, тип операции, время (московское время)
  • Логи организованы по дате запуска бота

Эти данные необходимы для:

  • Предотвращения мошенничества и дублирования платежей
  • Разрешения споров и технических проблем с зачислением
  • Соблюдения финансовой отчетности
  • Обеспечения безопасности и честности игровой экономики
